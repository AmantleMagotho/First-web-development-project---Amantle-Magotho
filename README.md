Project Overview
This project was developed as part of my Web & Multimedia Development module at Botswana Accountancy College. The goal was to design and build a brand-style website for Warner Bros that showcases movies, TV shows, games, and apps, while providing an engaging and accessible user experience.

The website serves as a platform for users to:
- Watch movie trailers and see release dates
- Explore video games and downloadable apps
- Learn more about Warner Bros history and subsidiaries
- Sign up for a newsletter to suggest features they want to see

Technologies Used and their purpose:
- HTML5:	Semantic structure and content
- CSS3:	Styling, layout, responsive design
- JavaScript:	Interactivity (dark/light mode, language selection) (was not able to fully explore)
- Visual Studio Code:	Code editor
- Lucid Chart:	Site mapping and information architecture
- Bootstrap Icons:	Icon library for buttons and UI elements

Key Features:
User Experience
Responsive Design – Works seamlessly on desktop, tablet, and mobile

Content
- Movie trailers with release date information
- Video game and app showcase
- Warner Bros company information and subsidiary details
- Newsletter signup for user feedback

Accessibility
- Clear navigation structure
- Perceivable and operable interface
- Robust design across different browsers and devices

Future Developments / Recommendations
The following features are planned for future iterations:
- User Account System – Allow users to create profiles and log in
- "Continue Watching" – Users can resume movies/shows from where they left off
- Viewing History – Track previously watched content
- Personalized Recommendations – Suggest content based on viewing history

What I Learned
- Planning matters – Creating a site map and page templates before coding saved time and reduced rework
- Responsive design is essential – Testing on real device sizes revealed issues I wouldn't have caught on desktop alone
- Accessibility features add value – Dark/light mode and language selection make the website usable for more people
- Documentation is a skill – Writing about technical work clearly is just as important as the code itself
